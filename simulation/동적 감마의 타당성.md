# Avellaneda-Stoikov 모델 확장: 동적 γ 및 Safety Cap

Avellaneda-Stoikov 모델의 확장 - 동적 감마와 best bid/ask 제약 추가(비정상적 호가 제출 상황 방지)


위험민감도는 경제적으로 의미를 가지는 변수이다. 
위험민감도가 크다는 것은 시장의 변화가 관측되었을 때, 그 시장의 변동성에 민감하게 반응하겠다는 뜻이다.
위험민감도가 작다는 것은 시장의 변화가 관측되었을 때, 그러한 움직임은 시장의 노이즈 정도로 해석하는 경향을 높인다는 뜻. 즉
내 의사결정에 시장의 변동을 많이 반영하지 않겠다.


이 때, A-S 논문의 식을 기반으로 해석을 하면 위험민감도의 크기에 따라 우리의 의사결정은 다음과 같이 달라진다.
r 이 크면? 
   최소 스프레드 term 및 spread 식의 크기가 커지고 이에따라, (T-t) 의 영향력이 커진다.
   rp 또한 감마* 변동성 * 재고 * (T-t) 의 식에 따라 (T-t) 의 영향력이 너무 커지고 재고의 변화에 따른
   민감도 또한 논문에서보다 확실히 커지게 된다.

r 이 작으면? 
   spread 식의 크기가 작아지고, (T-t) 의 영향력이 작아진다.
   스프레드는 최소 spread term 과 (T-t) term 에 의한 spread 가 있는데 
   감마가 커지면, 최소 spread term 은 어떻게 되는지 모르겠지만, 적어도 (T-t) 에 의한 영향력이 커져서
   스프레드가 장 초반에 넓고, 후반 갈 수록 줄어드는 역삼각혐 모양의 수렴을 보여준다.

1. γ의 동적 조정 (Time-Varying Risk Aversion)

1.1 배경: 원 논문의 한계

원 논문에서 위험 회피 계수 γ는 상수로 설정됩니다:

```
Reservation Price: r = s - q × γ × σ² × (T-t)
Optimal Spread: δ = γσ²(T-t) + (2/γ)ln(1 + γ/k)
```

그러나 γ를 상수로 두면 다음 문제가 발생합니다:

| γ 값 | 장 초반~중반 | 장 막판 |
|------|-------------|---------|
| 작은 γ (예: 0.1) | 적절한 스프레드 | 스프레드가 여전히 넓음 (체결 어려움) |
| 큰 γ (예: 5.0) | 스프레드 과도하게 넓음 | 적절한 스프레드 |

단일 γ로는 장 전체에 걸쳐 적절한 호가를 제시하기 어렵습니다.

1.2 제안: Time-Varying γ

```
γ(t) = {
    γ_base,                                        if time_remaining > threshold
    γ_base + α × (1 - time_remaining/threshold),   if time_remaining <= threshold
}
```

또는 Indicator Function을 활용한 단순화된 버전:

```
τ_eff = {
    threshold (상수),   if time_remaining > threshold   (장 초반~중반)
    time_remaining,     if time_remaining <= threshold  (장 막판)
}

r = s - q × γ × σ² × τ_eff
δ = γσ² × τ_eff + (2/γ)ln(1 + γ/k)
```

1.3 경제학적 정당성

(1) Deadline Effect (마감 효과)

행동경제학에서 마감이 가까워질수록 의사결정 패턴이 변화한다는 현상이 널리 알려져 있습니다. 트레이더는 장 마감 전 포지션 청산 압박으로 인해 위험 회피 성향이 증가합니다.

> "As deadlines approach, decision-makers exhibit systematic changes in risk preferences."

(2) Inventory Carrying Cost (재고 유지 비용)

잔고를 overnight으로 넘기면 다음과 같은 추가 비용/리스크가 발생합니다:
- 갭 리스크 (Gap Risk)
- 자금 조달 비용 (Funding Cost)
- 규제 자본 요건

장 막판에는 이러한 비용이 급격히 부각되어 청산 유인이 증가합니다.

(3) Time-Varying Preferences (시간 가변적 선호)

경제학에서 시간에 따른 선호도 변화는 널리 인정됩니다:
- Hyperbolic Discounting (쌍곡선 할인)
- State-Dependent Preferences (상태 의존적 선호)

위험 회피도 역시 상황(시간, 재고 수준 등)에 따라 변할 수 있다는 것은 이론적으로 합당합니다.

(4) 학술적 선례

| 연구 | 접근 방식 | 출처 |
|------|-----------|------|
| Alpha-AS | 강화학습으로 γ를 동적 조정 | PLOS ONE (2022) |
| ISAC | γ를 action space로 두고 (재고, 시간)에 따라 조정 | GitHub |
| Hummingbot | γ_max × IRA(Inventory Risk Aversion)로 스케일링 | Hummingbot Blog |

1.4 원 논문과의 관계

- 원 논문의 수식 구조는 유지 (RP, Spread 공식 동일)
- γ라는 파라미터의 값만 시간에 따라 조정
- 이론적 프레임워크를 깨지 않으면서 실무적 현실 반영

2. Safety Cap (호가 범위 제한)

2.1 배경: 극단적 상황에서의 비정상 호가

γ가 크거나 재고가 많이 쌓이면 다음과 같은 비정상적 상황이 발생할 수 있습니다:

| 상황 | 의미 | 문제 |
|------|------|------|
| LP Bid > Market Mid | 매수 호가가 중간가보다 높음 | 즉시 손실 체결 |
| LP Ask < Market Mid | 매도 호가가 중간가보다 낮음 | 즉시 손실 체결 |

예시 (γ = 5.0, q = 3, mid = 100):
```
RP = 100 - 3 × 5.0 × 4 × (1/6) = 90.0
Half Spread = 1.96
LP Bid = 90.0 - 1.96 = 88.04  (Market Mid = 100보다 12pt 낮음!)
```

2.2 제안: Safety Cap

```python
# 최종 호가에 cap 적용
bid_final = min(bid_raw, mid_price - min_distance)  # Bid ≤ Mid - ε
ask_final = max(ask_raw, mid_price + min_distance)  # Ask ≥ Mid + ε
```

ε = min_distance는 최소 호가 거리 (예: 0 또는 tick_size)

2.3 경제학적 정당성

(1) No-Arbitrage Condition (무차익 거래 조건을 제시하는 꼴)

Bid > Mid 또는 Ask < Mid는 즉시 차익거래 기회를 제공합니다:
- Bid > Mid: 누군가 LP에게 팔면 즉시 시장에서 더 싸게 살 수 있음
- Ask < Mid: 누군가 LP에게서 사면 즉시 시장에서 더 비싸게 팔 수 있음

합리적 시장 참여자는 이런 호가를 제시하지 않습니다.

(2) Rational Market Maker (합리적 마켓 메이커)

합리적인 LP는 손실이 확정된 호가를 제시하지 않습니다. Safety Cap은 이러한 합리성 제약(Rationality Constraint)의 구현입니다.

(3) Model Boundary Condition (모델 경계 조건)

수학적 모델은 극단적 파라미터에서 비현실적 결과를 낼 수 있습니다. Safety Cap은 모델의 호가 제시 유효 범위를 설정하는 것입니다.

이는 모델을 반박하는 게 아니라 보완하는 것입니다.

(4) 실무 관행

실제 거래소 및 마켓메이커는 호가 제출 전 Sanity Check를 수행합니다:

(5) Skewed Quote의 경제학적 해석

Safety Cap 적용 시 스프레드가 비대칭(Skewed)해질 수 있습니다:

| 재고 상태 | MM의 의도 | 제약 | 결과 |
|-----------|-----------|------|------|
| Long (q > 0) | Ask를 낮춰서 팔고 싶음 | Ask ≥ Mid + ε | Ask는 mid+ε에 고정, Bid만 낮게 → Ask 쪽 skew |
| Short (q < 0) | Bid를 높여서 사고 싶음 | Bid ≤ Mid - ε | Bid는 mid-ε에 고정, Ask만 높게 → Bid 쪽 skew |

이는 다음과 같이 해석됩니다:

1. 의무 호가 제약 (Market Maker Obligation)
   - 거래소가 MM에게 양방향 호가 의무 부과
   - 한쪽만 호가 철회 불가 → 비대칭 스프레드로 대응

2. Skewness = 재고 청산 의지의 표현
   - 체결시키고 싶은 방향: tight (mid에 가깝게)
   - 체결시키기 싫은 방향: wide (mid에서 멀게)

3. 시장에서 실제로 관찰되는 현상
   - 옵션 MM들이 델타 헷지 필요 시 skewed quote 제시
   - 이는 정보 노출이지만, 의무 호가 때문에 어쩔 수 없음

Safety Cap으로 인한 skewed quote는 "청산 욕구 vs 의무 호가 제약"의 트레이드오프를 반영하며 실무에서 실제로 관찰되는 현상과 일치합니다.

2.4 원 논문과의 관계

- 원 논문은 정상적인 파라미터 범위를 암묵적으로 가정
- Cap은 모델을 반박하는 게 아니라 보완하는 것
- 이론적 최적해가 실무적 제약을 위반할 때만 작동
