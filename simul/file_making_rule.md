문서 작성 공통 규칙

0. 목적
이 문서는 여러 MD 파일을 작성할 때 공통적으로 따라야 할 기준을 정의한다.
모든 문서는 이 규칙을 우선 기준으로 삼는다.

1. 강조 표기 사용 금지
문서 내에서 ##, ** 와 같은 마크다운 강조 표기는 사용하지 않는다.
과도한 강조, 시각적 장식, 불필요한 형식 강조보다 문장 자체의 명확성을 우선한다.
제목, 구분, 중요도 표현은 문장 구조와 문단 배치로 해결한다.

2. 인덱싱 원칙
문서의 큰 구조를 구분하기 위해 번호 체계 사용은 권장한다.

문서 구조상 하위 단계 구분이 꼭 필요한 경우에만 제한적으로 사용할 수 있다.
가능하면 하위 번호 대신 문단 구분, 짧은 소제목, 목록 형태로 구조를 정리한다.

3. 파일 간 역할 분리
각 MD 파일은 서로 다른 주제, 목적, 범위를 가져야 한다.
하나의 파일 안에 여러 역할을 섞지 않는다.
예를 들어 개념 설명 문서, 작업 절차 문서, 정책 문서, 예시 문서는 각각 분리한다.

4. 파일 간 내용 충돌 금지
각 MD 파일에 서술된 기준, 규칙, 정의는 다른 MD 파일과 충돌해서는 안 된다.
새 문서를 만들거나 기존 문서를 수정할 때에는 관련 문서와의 정의, 표현, 범위를 반드시 맞춘다.
서로 다른 문서에서 같은 대상을 설명한다면 서술 수준은 달라도 핵심 의미와 규칙은 일치해야 한다.

5. 중복 서술 최소화
여러 파일에 동일한 내용을 반복해서 적지 않는다.
어떤 규칙이나 정의가 이미 특정 MD 파일의 책임 범위에 포함되어 있다면, 다른 파일에서는 이를 반복 설명하지 않고 필요한 범위에서만 간단히 참조한다.
중복은 유지보수 비용과 충돌 가능성을 높이므로 최소화한다.

6. 문서 범위 명확화
각 MD 파일은 문서 첫 부분에서 자신이 다루는 범위를 분명히 해야 한다.
이 문서가 무엇을 설명하는지, 무엇은 다루지 않는지를 분명히 적는다.
범위가 불명확하면 다른 문서와의 중복이나 충돌이 발생할 수 있다.

7. 표현 원칙
문장은 간결하고 직접적으로 작성한다.
모호한 표현, 해석이 갈릴 수 있는 표현, 과장된 표현은 피한다.
규칙 문서는 가능한 한 명령형 또는 정의형 문장으로 작성한다.
예외가 허용되는 경우에는 그 조건을 분명히 적는다.

8. 문서 수정 원칙
기존 문서와 충돌 가능성이 있는 변경은 단일 문서만 수정하지 말고 관련 문서 전체 기준에서 검토한다.
새 기준을 추가할 때는 기존 기준을 대체하는지, 보완하는지, 예외를 만드는지 명확히 해야 한다.

9. 우선순위 원칙
문서 간 해석 차이가 발생하지 않도록 작성하는 것이 원칙이다.
다만 불가피하게 해석 우선순위가 필요하다면 공통 규칙 문서, 정책 문서, 개별 작업 문서 순으로 따른다.

10. 최종 목표
모든 MD 파일은 서로 독립적으로 읽히되, 전체적으로는 하나의 일관된 문서 체계를 이루어야 한다.
각 문서는 자신의 역할만 분명히 담당하고, 다른 문서와 겹치거나 충돌하지 않아야 한다.
