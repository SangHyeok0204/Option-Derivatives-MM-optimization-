# 프로젝트 규칙

## 문서 동기화 규칙

simulation 폴더 내 코드(.py, .ipynb)를 수정할 때 다음을 수행한다.

1. 변경 내용이 기존 md 파일의 설명과 충돌하는지 확인한다.
2. 충돌하면 해당 md 파일도 함께 업데이트한다.
3. simulation/file_making_rule.md 규칙을 따른다.

## 문서 역할 분담

프로젝트가이드.md는 프로젝트 전체 철학, 목적, 방향을 다룬다.
paper모드.md는 Paper 모드 설정과 수식을 다룬다.
develop모드.md는 Develop 모드 설정과 변경점을 다룬다.
디벨롭 타당성.md는 확장의 경제학적 정당성을 다룬다.
데이터_수집_설계.md는 크롤링 데이터 수집 설계를 다룬다.

## 문서 작성 규칙 요약

마크다운 강조 표기(##, **)는 사용하지 않는다. 단, 이 CLAUDE.md 파일은 예외로 한다.
번호 체계는 1., 2., 3. 형태를 사용한다.
하위 번호(1.1, 1.2)는 꼭 필요한 경우에만 사용한다.
중복 서술을 최소화한다.
문서 첫 부분에 범위를 명확히 한다.

## 코드 수정 시 체크리스트

코드에서 새로운 파라미터를 추가했는가? 해당 모드 md 파일에 반영한다.
코드에서 수식을 변경했는가? 해당 모드 md 파일에 반영한다.
코드에서 새로운 기능을 추가했는가? 적절한 md 파일에 반영한다.
데이터 수집 관련 변경인가? 데이터_수집_설계.md에 반영한다.
